LogicBlocks supports serialization through the `AutoBlock` base class, available in the `Chickensoft.LogicBlocks.Auto` package. `AutoBlock` builds on top of the Chickensoft [Introspection] and [Serialization] packages to provide automatic state pre-allocation and save/load support.

<Callout type="note">
The serialization system works alongside System.Text.Json, using metadata generated by the Introspection generator to describe types and their properties — enabling it to work on ahead-of-time (AOT) platforms like iOS.
</Callout>

### 🔧 Setup

Call `LogicBlockSerialization.Setup()` once at startup to register the required JSON converters with the Chickensoft Serialization system:

```csharp
using Chickensoft.LogicBlocks.Auto;

LogicBlockSerialization.Setup();
```

### 💽 Defining a Serializable Logic Block

To make a logic block serializable, extend `AutoBlock` instead of `LogicBlock`, give it an `[Id]`, and use `Preallocate<TBaseState>()` to register states automatically via introspection.

```csharp
using Chickensoft.Introspection;
using Chickensoft.LogicBlocks;
using Chickensoft.LogicBlocks.Auto;

[Meta, Id("my_logic")]
public partial class MyLogic : AutoBlock
{
  public MyLogic()
  {
    Preallocate<MyState>();
  }

  // Implement GetSaveData to define what gets stored when saving.
  public override ILogicBlockSaveData GetSaveData(LogicBlockData data) =>
    new MyLogicSaveData { Data = data };
}

// Save data record must also be identifiable.
[Meta, Id("my_logic_save_data")]
public partial class MyLogicSaveData : ILogicBlockSaveData
{
  [Save("data")]
  public required LogicBlockData Data { get; init; }
}
```

States must be introspective (`[Meta]`) and identifiable (`[Id]`). Non-abstract states need an `[Id]`.

```csharp
[Meta, StateDiagram]
public abstract partial record MyState : LogicBlockState
{
  [Meta, Id("my_state_off")]
  public partial record Off : MyState;

  [Meta, Id("my_state_on")]
  public partial record On : MyState;
}
```

<Callout type="caution">
Serializable type `[Id]`s need to be globally unique across all serializable types used in your project.
</Callout>

### 🤐 Saving a Logic Block

Use `logic.Save()` to get a snapshot of the current state, blackboard, and history:

```csharp
var logic = new MyLogic();
logic.Start<MyState.Off>();

// Get save data (can be serialized to JSON)
var saveData = logic.Save();
```

You can serialize `saveData` to JSON using the Chickensoft Serialization system:

```csharp
using Chickensoft.Serialization;
using Chickensoft.LogicBlocks.Auto;
using System.Text.Json;

LogicBlockSerialization.Setup();

var options = new JsonSerializerOptions {
  WriteIndented = true,
  TypeInfoResolver = new SerializableTypeResolver(),
  Converters = { new SerializableTypeConverter() }
};

var json = JsonSerializer.Serialize(saveData, options);
```

### 📂 Loading a Logic Block

To restore a logic block from saved data, deserialize the save data and call `Start(LogicBlockData)`:

```csharp
// Deserialize the save data
var saveData = JsonSerializer.Deserialize<MyLogicSaveData>(json, options);

// Create a fresh logic block and load from saved data
var logic = new MyLogic();
logic.Start(saveData!.Data);

// The logic block is now in the saved state.
```

If you need to load into an _existing_ logic block (to preserve existing bindings), stop it first:

```csharp
// Stop the existing logic block
existingLogic.Stop();

// Reload from saved data
existingLogic.Start(saveData!.Data);
```

### 🔄 OnLoad Callback

Override `OnLoad()` to run code after the logic block is restored from saved data:

```csharp
[Meta, Id("my_logic")]
public partial class MyLogic : AutoBlock
{
  // ...

  public override void OnLoad()
  {
    // Called after the logic block has been loaded from saved data.
    // Great for re-establishing connections or validating loaded state.
  }
}
```

### 💾 Persisting Blackboard Values

By default, blackboard values are treated as runtime dependencies and are not serialized. Call `Save<TData>(factory)` to opt a value in to persistence. LogicBlocks will serialize it when saving and automatically restore it to the blackboard when loading. The factory provides a default value if none is found in the save data.

```csharp
[Meta, Id("my_related_data")]
public partial record MyRelatedData
{
  public required string Name { get; init; }
  public string? Description { get; init; }
}

var logic = new MyLogic();

// Not persisted — just adding a runtime dependency.
logic.Set(new MyRelatedService());

// Persisted automatically on save and restored on load.
logic.Save(() => new MyRelatedData { Name = "default" });
```

### 📦 Embedding Data in Save Records

For more control over what gets saved, override `GetSaveData()` and pull values directly from the blackboard to include in your save record. This is useful when you want to transform or selectively include data rather than persisting a blackboard value as-is.

```csharp
[Meta, Id("my_logic_save_data")]
public partial class MyLogicSaveData : ILogicBlockSaveData
{
  [Save("data")]
  public required LogicBlockData Data { get; init; }

  [Save("score")]
  public required int Score { get; init; }
}

[Meta, Id("my_logic")]
public partial class MyLogic : AutoBlock
{
  public override ILogicBlockSaveData GetSaveData(LogicBlockData data) =>
    new MyLogicSaveData {
      Data = data,
      Score = Get<MyGameData>().Score
    };
}
```

### ⏳ Versioning

LogicBlocks supports [versioning] states through the Chickensoft Serialization system:

```csharp
[Meta, Id("versioned_state")]
public abstract partial record VersionedState : LogicBlockState;

[Meta, Version(1)]
public partial record Version1 : VersionedState;

[Meta, Version(2)]
public partial record Version2 : VersionedState;
```

### 😶‍🌫️ Test States

When unit testing, you may create a concrete implementation of an abstract state. If your logic block is an `AutoBlock`, it will try to preallocate all concrete states — including test ones. Add `[TestState]` to exclude a state from preallocation:

```csharp
[TestState, Meta, Id("my_test_state")]
public partial class MyTestState : MyAbstractState;
```
