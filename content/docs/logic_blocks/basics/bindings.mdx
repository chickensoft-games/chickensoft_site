LogicBlocks provides a binding system that allows objects to listen to a particular logic block. Bindings allow you to observe inputs, outputs, state changes, and lifecycle events declaratively.

## 📥 Observing Inputs

```csharp
using var binding = logic.Bind();

binding.OnInput((in MyLogicBlock.Input.SomeInput input) => {
  // Observe a particular input.
});
```

## 📤 Observing Outputs

```csharp
using var binding = logic.Bind();

binding.OnOutput((in MyLogicBlock.Output.SomeOutput output) => {
  // Handle a particular output.
});
```

## 🔄 Observing State Changes

```csharp
using var binding = logic.Bind();

binding.OnState((MyLogicBlockState.SomeState state) => {
  // Respond to a state change. This is only called when the logic block
  // transitions to the specified state type.
});
```

## 🔁 Observing Lifecycle Events

```csharp
using var binding = logic.Bind();

binding.OnStart(() => {
  // Called when the logic block is started.
});

binding.OnStop(() => {
  // Called when the logic block is stopped.
});

binding.OnLoad(() => {
  // Called when the logic block is started with a LogicBlockData type.
});
```

## 🔗 Chaining

All binding methods return the binding itself, so you can chain them together.

```csharp
using var binding = logic.Bind()
  .OnOutput((in MyLogicBlock.Output.SomeOutput output) => HandleOutput(output))
  .OnState((MyLogicBlockState.SomeState state) => HandleState(state))
  .OnStart(() => HandleStart())
  .OnStop(() => HandleStop());
```
