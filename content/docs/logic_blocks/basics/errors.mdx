In LogicBlocks, exceptions thrown inside input handlers propagate naturally — they are not caught internally. This gives you full control over error handling.

## 🚨 Handling Errors

If a state's input handler may throw, it will propagate out through `logic.Input()`:

```csharp
public record MyState : MyBaseState, IGet<Input.SomethingRisky> {
  public Type On(in Input.SomethingRisky input) {
    // This exception will propagate to whoever called logic.Input().
    throw new InvalidOperationException("Something went wrong!");
  }
}
```

```csharp
try {
  logic.Input(new MyLogicBlock.Input.SomethingRisky());
}
catch (Exception e) {
  // Handle or log the error.
  Console.WriteLine($"Error: {e.Message}");
}
```
