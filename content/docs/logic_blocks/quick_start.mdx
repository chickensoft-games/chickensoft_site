## 💡 Creating a Logic Block

A logic block is simply a class that extends `LogicBlock`. Logic blocks receive inputs, maintain a single state value, and produce outputs.

```csharp
public partial class LightSwitch : LogicBlock {
  // Register all possible state instances in the constructor.
  public LightSwitch() {
    Set(new LightSwitchState.PoweredOff());
    Set(new LightSwitchState.PoweredOn());
  }

  // Optionally override GetInitialState to specify the starting state.
  // If omitted, you must call Start<TState>() explicitly.
  public override Type GetInitialState() => typeof(LightSwitchState.PoweredOff);
}

// The [StateDiagram] attribute on the base state type enables diagram generation.
[StateDiagram]
public abstract record LightSwitchState : LogicBlockState {
// Substates are sometimes nested inside their parent states to help
// organize the code.

  // By convention, inputs are defined in a static nested class called Input.
  public static class Input {
    public readonly record struct Toggle;
  }

  // By convention, outputs are defined in a static nested class called Output.
  public static class Output {
    public readonly record struct StatusChanged(bool IsOn);
  }

  // To reduce unnecessary heap allocations, inputs and outputs should be
  // readonly record structs.

  // On state.
  public record PoweredOn : LightSwitchState, IGet<Input.Toggle> {
    public PoweredOn() {
      // Announce that we are now on.
      this.OnEnter(() => Output(new Output.StatusChanged(IsOn: true)));
    }

    public Type On(in Input.Toggle input) => To<PoweredOff>();
  }

  // Off state.
  public record PoweredOff : LightSwitchState, IGet<Input.Toggle> {
    public PoweredOff() {
      // Announce that we are now off.
      this.OnEnter(() => Output(new Output.StatusChanged(IsOn: false)));
    }

    public Type On(in Input.Toggle input) => To<PoweredOn>();
  }
}
```

To make a logic block, you simply extend the `LogicBlock` class, register each state instance in the constructor using `Set()`, and implement the `IGet<TInput>` interface on states for each input they handle. Input handlers return `Type` (the type of the next state) using the `To<TState>()` helper.

## 🔌 Using a Logic Block

LogicBlocks includes a simple binding system to enable you to write declarative code, even in imperative environments like a game engine. Bindings allow you to monitor the inputs a state machine receives and the outputs it produces, in addition to its state changes and lifecycle events.

```csharp
var logic = new LightSwitch();

// Start the logic block to activate the initial state.
logic.Start();

// Add an input to turn our light switch on.
logic.Input(new LightSwitchState.Input.Toggle());

// The logic block's State represents the current state.
var state = logic.State; // PoweredOn

// Bindings allow you to observe the logic block easily.
using var binding = logic.Bind();

// Monitor an output:
binding.OnOutput((in LightSwitchState.Output.StatusChanged output) =>
  Console.WriteLine(
    $"Status changed to {(output.IsOn ? "on" : "off")}"
  )
);

// Can also use bindings to monitor inputs, state changes, and lifecycle events.
//
// In general, prefer monitoring outputs over state changes for more
// flexible code.

// Monitor an input:
binding.OnInput((in LightSwitchState.Input.Toggle input) =>
  Console.WriteLine("Toggled!")
);

// Monitor a specific type of state:
binding.OnState((LightSwitchState.PoweredOn _) =>
  Console.WriteLine("Powered on!")
);

// Monitor lifecycle events:
binding.OnStart(() => Console.WriteLine("Logic block started!"));
binding.OnStop(() => Console.WriteLine("Logic block stopped!"));
```
