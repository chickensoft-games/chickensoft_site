To start off the tutorial, we'll start by making sure we have the right dependencies and then setup an empty logic block.

<Callout type="caution">
This tutorial assumes a high degree of familiarity with state machines and C#. If you're new to either, you may wish to [learn more about C#][learn csharp] or read the basics on [statecharts].
</Callout>

### 📦 Environment Setup

If you haven't already, make sure you've [installed everything]. You'll need LogicBlocks. Feel free to install the optional diagram generator so you can [see your code as a state diagram][state diagram].

### 📝 Creating a Logic Block

To define a logic block, we make a class that extends `LogicBlock`. By convention, we define our state types near the logic block, often in the same file or a companion file.

Let's go ahead and make an empty logic block for the timer.

```csharp
public partial class TimerLogic : LogicBlock {
  public TimerLogic() {
    // Register all possible state instances on the blackboard.
    Set(new TimerState.PoweredOff());
  }

  // Define the starting state.
  public override Type GetInitialState() => typeof(TimerState.PoweredOff);
}

[StateDiagram]
public abstract record TimerState : LogicBlockState {

  public static class Input;

  public static class Output;

  public record PoweredOff : TimerState;
}
```

We've created an empty abstract base state, a starting state `PoweredOff`, and two empty static classes which will store our input and output types. We've also specified that our initial state should be `PoweredOff` by overriding `GetInitialState()`.

<Callout type="note">
Each state must be registered on the blackboard using `Set()` in the logic block's constructor. This pre-allocates state instances so the logic block doesn't need to create them during transitions, preventing heap allocations at runtime.
</Callout>

<Callout type="tip">
If you use the [Chickensoft Introspection] generator and the `AutoBlock` base class (from `Chickensoft.LogicBlocks.Auto`), states can be preallocated automatically using `Preallocate<TBaseState>()`. This prevents human error from forgetting to register a state, especially as you refactor.
</Callout>
