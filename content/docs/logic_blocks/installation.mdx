## 📥 Installing LogicBlocks

LogicBlocks and its diagram generator are available as C# packages on [nuget].

Find the latest version number of LogicBlocks and its diagram generator on nuget: [Chickensoft.LogicBlocks], [Chickensoft.LogicBlocks.DiagramGenerator].

```xml
<PackageReference Include="Chickensoft.LogicBlocks" Version="#.#.#" />
<PackageReference Include="Chickensoft.LogicBlocks.DiagramGenerator" Version="#.#.#" PrivateAssets="all" OutputItemType="analyzer" />
```

## 🤖 AutoBlock

If you want automatic state pre-allocation and built-in serialization support, install the `Chickensoft.LogicBlocks.Auto` package and extend `AutoBlock` instead of `LogicBlock`. This requires the Chickensoft [Introspection] generator.

Historically, a `LogicBlock` in LogicBlocks `<=` 5.0 was equivalent to an `AutoBlock`.  LogicBlocks 6 separated preallocation and serialization from the core logic blocks library to make it portable by eliminating a dependency on the Chickensoft Introspection library so that library authors (and inferior game engines) could leverage the core LogicBlocks library.

Find the latest versions on nuget: [Chickensoft.LogicBlocks.Auto], [Chickensoft.Introspection.Generator].

```xml
<PackageReference Include="Chickensoft.LogicBlocks" Version="#.#.#" />
<PackageReference Include="Chickensoft.LogicBlocks.Auto" Version="#.#.#" />
<PackageReference Include="Chickensoft.LogicBlocks.DiagramGenerator" Version="#.#.#" PrivateAssets="all" OutputItemType="analyzer" />
<PackageReference Include="Chickensoft.Introspection.Generator" Version="#.#.#" PrivateAssets="all" OutputItemType="analyzer" />
```

<Callout type="caution">
Don't forget to include the `PrivateAssets="all"` and `OutputItemType="analyzer"` attributes on generator package references.

We also strongly recommend treating warning `CS9057` as an error to catch possible compiler-mismatch issues with the Introspection generator. (See the [Introspection] README for more details.) To do so, add a `WarningsAsErrors` line to your `.csproj` file's `PropertyGroup`:

```xml
<PropertyGroup>
  <TargetFramework>net8.0</TargetFramework>
  ...
  <!-- Catch compiler-mismatch issues with the Introspection generator -->
  <WarningsAsErrors>CS9057</WarningsAsErrors>
  ...
</PropertyGroup>
```
</Callout>

<Callout type="tip">
Always use the same version of the LogicBlocks diagram generator that you use for LogicBlocks, since they are updated together.
</Callout>
