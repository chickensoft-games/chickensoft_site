Additional functionality and troubleshooting tips for LogicBlocks are described here.

## 📐 Generic Logic Blocks

A type is generic in C# if it accepts type parameters or is nested inside a type that does. Logic blocks can exist inside generic types, but there is one important limitation: `AutoBlock`'s `Preallocate<T>()` relies on the [Introspection] generator, which does not support generic types. This means:

- 🚫 ✨ States nested inside a generic type cannot be discovered by `Preallocate<T>()`. You must register them manually with `Set()` in the constructor.

- 🚫 💾 Logic blocks in generic contexts cannot use `AutoBlock` and are therefore not serializable, since the [Serialization] system is built on top of [Introspection].

If your logic block is in a generic context, extend `LogicBlock` directly and register states in the constructor yourself:

```csharp
public class MyGenericType<T> {
  public class MyLogicBlock : LogicBlock {
    public abstract record State : LogicBlockState { }
    public record StateOne : State;
    public record StateTwo : State;

    public override Type GetInitialState() => typeof(StateOne);

    public MyLogicBlock() {
      Set(new StateOne());
      Set(new StateTwo());
    }
  }
}
```

## 🔔 Logic Block Subscriptions

Use `OnStartSubscriptions()` to subscribe to services or reactive dependencies when the logic block starts, and have them automatically cleaned up when it stops:

```csharp
public class MyLogicBlock : LogicBlock {
  public override IEnumerable<IDisposable> OnStartSubscriptions()
  {
    // Return IDisposable subscriptions — they are disposed automatically on Stop().
    yield return Get<IMyService>().SomeEvent.Subscribe(OnSomethingHappened);
  }

  private void OnSomethingHappened(SomeEventArgs args)
  {
    // Route the event as an input to whichever state is currently active.
    Input(new MyState.Input.SomethingHappened(args.Value));
  }
}
```

This is the recommended way to subscribe to events that should be active for the entire lifetime of the logic block. For state-specific subscriptions, use `this.OnEnter` and `this.OnExit` inside the state's constructor instead.

## ⏫ State History

LogicBlocks supports a history stack that states can use to implement a pushdown automaton — i.e., returning to a previous state after some interruption.

States can push their type onto the history stack and later pop it to return to that state:

```csharp
public record InterruptedState : MyState, IGet<MyState.Input.Resume> {
  public InterruptedState() {
    this.OnEnter(() => Push()); // Push current state type onto history
  }

  public Type On(in MyState.Input.Resume input) {
    // Pop and return to the previous state
    var previousState = Pop();
    return previousState ?? To<MyState.Idle>();
  }
}
```

The history stack has a configurable max capacity (default: 8 entries). Configure it in the logic block constructor:

```csharp
public MyLogicBlock() : base(maxHistory: 16) { }
```
